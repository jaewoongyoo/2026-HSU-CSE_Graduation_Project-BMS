# AI 작업 규칙

이 문서는 `ai/` 작업 공간에서 사용하는 디렉터리별 규칙을 한 곳에 모아둔 문서입니다.

기본 원칙:

- 구조 소개와 진입점 설명은 `ai/README.md`를 먼저 봅니다.
- 세부 운영 규칙은 이 문서를 기준으로 합니다.
- 규칙 문서를 여러 디렉터리에 흩뿌리지 않고, 가능한 한 이 디렉터리에서 관리합니다.

## 1. data/

`data/`는 데이터 저장 공간입니다.

- `external/`: NASA, CALCE, Oxford 같은 공개 원본 데이터셋
- `raw/`: 공통 로컬 구조로 복사하거나 압축 해제한 원본
- `interim/`: 데이터 정합화 과정의 중간 산출물
- `processed/`: 학습/평가에 바로 사용할 수 있는 최종 테이블 또는 시퀀스

규칙:

- 대용량 데이터는 Git에 커밋하지 않습니다.
- `external/`, `raw/`의 원본은 가급적 수정하지 않습니다.
- 수작업 편집보다 재현 가능한 변환 코드를 우선합니다.

## 2. artifacts/

`artifacts/`는 생성 결과물을 보관하는 공간입니다.

- `checkpoints/`: 학습 체크포인트
- `exports/`: export한 모델, scaler, 직렬화 메타데이터
- `reports/`: 평가 요약, 그래프, 실험 결과

규칙:

- 생성물은 소스 코드와 분리해서 관리합니다.
- 최종 배포용 모델 파일과 실험 중간 산출물을 구분합니다.
- 결과 파일 이름에는 날짜, 실험명, 버전 정보를 포함하는 것을 권장합니다.

## 3. configs/

`configs/`는 데이터셋, 전처리, 학습 관련 설정 파일을 두는 공간입니다.

권장 분류:

- `datasets/`: 데이터셋별 파싱 규칙과 스키마 설정
- `preprocessing/`: 리샘플링, 윈도잉, 정규화, 관측 공간 가정
- `models/`: LSTM 구조와 최적화 설정
- `experiments/`: 실험 단위 preset

## 4. scripts/

`scripts/`는 실행 스크립트와 일회성 유틸리티를 두는 공간입니다.

예시:

- 데이터셋 다운로드 스크립트
- 데이터 변환 진입점
- 학습 실행 스크립트
- 평가 및 export 스크립트

현재 `start_server.bat`, `test_api.py`는 기존 서비스용 보조 파일이며, 필요하면 이후에 `scripts/`로 정리할 수 있습니다.

## 5. notebooks/

`notebooks/`는 탐색적 분석용 공간입니다.

규칙:

- 노트북은 가볍고 임시적인 성격으로 유지합니다.
- 재사용할 로직은 `soh_service/` 아래의 적절한 패키지로 옮깁니다.
- 전처리나 학습 파이프라인의 기준 구현을 노트북에 두지 않습니다.

## 6. tests/

`tests/`는 AI 파이프라인 검증용 공간입니다.

예상 범위:

- 데이터셋 파서 테스트
- 전처리 검증 테스트
- 시퀀스 생성 테스트
- 추론 계약 검증 테스트

## 7. soh_service/

코드 구조는 `soh_service` 중심으로 가져갑니다.

- `api/`: FastAPI 라우터
- `core/`: 현재 공용 추론 로직
- `datasets/`: 데이터셋 로더/파서
- `preprocessing/`: 스마트폰 관측 공간 변환
- `training/`: 학습 및 평가 코드
- `inference/`: 추론 보조 유틸리티
- `utils/`: 공용 유틸리티
- `model/`: 현재 배포 모델 파일

규칙:

- 학습용 코드와 배포용 추론 코드는 같은 패키지 안에 두되 역할은 분리합니다.
- 새 AI 로직은 우선 `soh_service/` 아래의 적절한 하위 패키지에 추가합니다.
- 데이터나 아티팩트 같은 비코드 자산은 `soh_service/` 안에 두지 않습니다.
